Render Strategy

Choose mode based on reliability, control, and compatibility.

SharePointFileContent (recommended)

Reads HTML via SharePoint API and renders as inline iframe content.

Best for report bundles and anti-download behavior

DirectUrl

Loads target directly with iframe src. Lighter, but fewer controls.

Good for simple, trusted pages
contentDeliveryMode: "SharePointFileContent" htmlSourceMode: "FullUrl" fullUrl: "<configured-html-target>"

Core Features

Deep-linking: current subpage stored in ?uhvPage=....
Back/Forward support: browser history replays inline report states.
Link interception: approved extensions stay inside UHV host.
Scroll stabilization: protects host-page position during hydration.
Open in new tab: optional fallback for raw content view.

Deep-Link Contract and Browser Navigation

UHV keeps the host page URL as the source of truth for current dashboard view.

1
User opens the host page with deep-link query state and UHV resolves requested target.
2
Link clicks inside the dashboard are intercepted and pushed into host history state.
3
Back/Forward events restore previous report pages inline without leaving host page.
Host page URL + encoded deep-link state

Security Modes

Request targets are validated before loading.

StrictTenant Only URLs inside current tenant/site policy. Default Safe
Allowlist Only configured hosts and path prefixes. Controlled
AnyHttps Broadest mode for trusted scenarios. Use Carefully
securityMode: "StrictTenant" sandboxPreset: "Relaxed" iframeLoadTimeoutSeconds: 10
